# Performance Benchmark

This document describes a synthetic realistic-volume benchmark for VulcansTrace. All data is generated -- no real network traffic or firewall logs are used.

## Benchmark Setup

### Tool

A PowerShell script generates the synthetic log file:

```
tools/Generate-SyntheticFirewallLog.ps1
```

### Dataset

The synthetic log contains approximately 50,000 lines of Windows Firewall log entries with embedded attack patterns:

| Pattern | Lines | Description |
|---|---|---|
| Port scan | 150 | Scanner hitting 150 ports on a server in 5 minutes |
| Beaconing | 24 | C2 callback every 300 seconds for 2 hours |
| Lateral movement | 10 | Internal host connecting to admin ports on 10 internal servers |
| Flood | 500 | High-volume connection burst (500 events in 5 seconds) |
| Policy violations | 8 | Outbound connections on disallowed ports (Telnet, FTP) |
| Novelty | 5 | One-time connections to unique external IPs |
| Background noise | ~49,300 | Normal mixed inbound/outbound traffic |

### Machine Specs

| Parameter | Value |
|---|---|
| OS | Windows 11 |
| CPU | AMD Ryzen 5 3500 |
| RAM | 16 GB |
| .NET version | 9.0 |
| Configuration | Release |
| Disk | SSD |

### How To Reproduce

```powershell
# Step 1: Generate the synthetic log
.\tools\Generate-SyntheticFirewallLog.ps1 -TotalLines 50000 -OutputPath .\benchmark-pfirewall.log

# Step 2: Build in Release
dotnet build VulcansTrace.sln --configuration Release --no-restore

# Step 3: Run the WPF app, load the benchmark log, select High intensity, run analysis

# Step 4: Note the timing from the UI
```

## Limitations

- All data is synthetic and generated by a script. Real-world firewall logs may have different field distributions, encoding issues, or missing fields that could affect performance.
- The benchmark measures single-run performance on a single machine. Results will vary by hardware.
- The analysis is single-threaded (detectors run sequentially).
- The WPF UI adds overhead (rendering, data binding). The core engine performance may be faster when called directly.

## Repeatability

The synthetic log generator uses a deterministic random seed (42), so the same input is produced every run. To reproduce:

1. Run `Generate-SyntheticFirewallLog.ps1` with default parameters
2. Load the output in VulcansTrace
3. Select High intensity profile
4. Run analysis
5. Compare findings against the expected patterns above
